    How often should it refresh?
    Practical guidance for GitHub Actions + AviationWeather (AWC) Data API.
    • Hard limit: GitHub Actions scheduled workflows can run no more frequently than every 5 minutes (and may drift).
    • AWC guidance: keep requests limited in scope and frequency; rate limiting may apply. For large/global pulls, prefer cache files.
    • Recommendation for this monitor (≈159 stations): every 10 minutes is conservative; every 5 minutes is the practical max if you really need near real‑time SPECI detection.
